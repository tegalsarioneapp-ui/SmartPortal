You are a senior backend engineer. Fix the production crash causing HTTP 500 on `/api/health` in a Railway Node.js app.

## PROBLEM

* `/api/health` returns HTTP 500
* Server is running but crashes during request handling
* Likely caused by database or blocking logic

---

## REQUIRED FIXES (DO ALL)

### 1. NEVER crash health endpoint

Rewrite `/api/health` to ALWAYS return 200:

```js
app.get("/api/health", async (req, res) => {
  try {
    let dbOk = false;

    try {
      if (pool) {
        await pool.query("SELECT 1");
        dbOk = true;
      }
    } catch (err) {
      console.error("[Health] DB check failed:", err.message);
    }

    res.json({
      status: "ok",
      db: dbOk
    });

  } catch (err) {
    console.error("[Health] Fatal error:", err);
    res.json({
      status: "ok",
      db: false
    });
  }
});
```
